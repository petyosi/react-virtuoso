The table accepts data in two forms:

- a flat array inside `data: { data, groups: [] }`
- a grouped structure where `groups` marks the indexes of inserted group rows

That second form is what powers category headers, multi-level hierarchies, and sectioned dashboards.

## Targeted example

```tsx live
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader, GroupHeaderCell } from '@/components/ui/data-table'

const tableData = {
  data: [
    { label: 'Office (2)' },
    { id: 'SKU-001', name: 'Standing Desk', category: 'Office' },
    { id: 'SKU-002', name: 'Desk Lamp', category: 'Office' },
    { label: 'Peripherals (2)' },
    { id: 'SKU-003', name: 'USB-C Dock', category: 'Peripherals' },
    { id: 'SKU-004', name: 'Mechanical Keyboard', category: 'Peripherals' },
  ],
  groups: [
    { index: 0, level: 0 },
    { index: 3, level: 0 },
  ],
}

export default function App() {
  return (
    <DataTable className="rounded-xl" data={tableData} style={{ height: 320 }}>
      <GroupHeaderCell className="bg-muted px-3 py-2 font-medium">{({ row }) => row.data.label}</GroupHeaderCell>
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="category">
        <DataTableColumnHeader>Category</DataTableColumnHeader>
        <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Patterns

- Group rows can carry a separate type from regular rows.
- Keep group labels lightweight. They are still virtualized rows.
- Use the `level` value to style nested groups differently.
