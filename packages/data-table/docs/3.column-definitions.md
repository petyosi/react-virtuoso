After binding data, declare columns. Without column definitions the table has rows, but no grid to
render.

`DataTableColumn` selects a field. `DataTableColumnHeader` renders the header. `DataTableCell`
renders the field value or the full row.

## Basic columns

```tsx live
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'

const rows = [
  { id: 'SKU-001', name: 'Standing Desk', category: 'Office', price: 699, stock: 14 },
  { id: 'SKU-002', name: 'USB-C Dock', category: 'Peripherals', price: 229, stock: 42 },
  { id: 'SKU-003', name: 'Mechanical Keyboard', category: 'Peripherals', price: 169, stock: 28 },
]

const currency = new Intl.NumberFormat('en-US', {
  currency: 'USD',
  style: 'currency',
})

export default function App() {
  return (
    <DataTable className="rounded-xl" data={{ data: rows, groups: [] }} style={{ height: 280 }}>
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell>
          {({ row }) => (
            <div className="flex flex-col">
              <span className="font-medium">{row.data.name}</span>
              <span className="text-xs text-muted-foreground">{row.data.id}</span>
            </div>
          )}
        </DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="price">
        <DataTableColumnHeader className="justify-end">Price</DataTableColumnHeader>
        <DataTableCell className="text-right tabular-nums">{({ cellValue }) => currency.format(Number(cellValue))}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="stock">
        <DataTableColumnHeader>Status</DataTableColumnHeader>
        <DataTableCell>
          {({ row }) => (
            <span className={row.data.stock < 20 ? 'font-medium text-amber-600' : 'text-foreground'}>
              {row.data.stock < 20 ? 'Low stock' : 'Healthy'}
            </span>
          )}
        </DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Render params

Use `cellValue` when a cell only needs the selected field. Use `row.data` when formatting depends on
other fields or when the row can be either a data row or a group row.

```tsx
<DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>

<DataTableCell>
  {({ row }) => (
    <span className={row.data.stock < 20 ? 'text-amber-600' : undefined}>
      {row.data.stock < 20 ? 'Low stock' : 'Healthy'}
    </span>
  )}
</DataTableCell>
```

Common render params:

- `CellRenderParams` gives cells `cellValue`, `columnKey`, `column`, `columnState`, `row`, and
  `overlaidByScrollbar`.
- `ColumnHeaderRenderParams` gives headers `columnKey`, `column`, `columnState`, and
  `overlaidByScrollbar`.
- `GroupHeaderRenderParams` gives group headers `level` and `row`.
- `ColumnGroupHeaderRenderParams` gives column-group headers the group metadata.
- `HeaderSlotRenderParams` gives header slots the column metadata, column state, header ref, and
  scrollbar overlay state.

Use the specific exported type when you extract a renderer into a named function.

```tsx
import type { CellRenderParams } from '@virtuoso.dev/data-table'

function StockCell({ row }: CellRenderParams) {
  return <span>{row.data.stock < 20 ? 'Low stock' : 'Healthy'}</span>
}
```

## Context

Use the table `context` prop for state that table infrastructure needs but that does not belong in
each row: empty-state copy, row component styling flags, sticky wrapper settings, custom scroll
element state, or row-key prefixes.

```tsx
function EmptyPlaceholder({ context }) {
  return <div>{context.emptyMessage}</div>
}

;<DataTable
  EmptyPlaceholder={EmptyPlaceholder}
  computeRowKey={({ data, context }) => `${context.tenantId}:${data.id}`}
  context={{ emptyMessage: 'No archived products', tenantId: 'acme' }}
  data={{ data: rows, groups: [] }}
>
  {/* columns */}
</DataTable>
```

The same `context` value is passed to `computeRowKey`, `EmptyPlaceholder`, loading slots, row
component overrides, sticky wrapper overrides, and custom scroll elements. Cell and header render
functions do not receive `context`; close over app state directly when a renderer needs screen-local
values.

## Column identity

The `field` value is the public identity of a column. Persistence adapters store column width, order,
and visibility by field name so state can survive remounts and partially matching schemas.

Use stable field names, even when the column label is localized or restyled. If you generate columns
from model data, keep the generated field order stable; see
[Inferring Columns From Model Data](/data-table/inferring-columns-from-model-data/).

## Row identity

Use `computeRowKey` when rows have stable IDs. It receives the row data, row index, and table
context, and should return the React key for that row.

```tsx
<DataTable computeRowKey={({ data }) => data.id} data={{ data: rows, groups: [] }}>
  {/* columns */}
</DataTable>
```

Stable row keys prevent unnecessary remounts when sorting, filtering, grouping, or replacing the
data array. If grouped data mixes data rows and group rows, make sure group rows also return stable
keys.
