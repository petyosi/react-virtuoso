---
title: Keeping Column and Row State Stable
description: Avoid lost column settings and unnecessary row remounts by keeping column keys, row keys, and table context stable.
sidebar:
  label: Stable State
---

Column and row identity decide whether the table can preserve user state. If a column's identity
changes, saved width, order, sticky, and visibility state for that column can no longer match. If a
row key changes, React remounts that row, which can reset focus, local cell state, and expensive
child components.

Keep identities stable unless you intentionally want the table to treat a column or row as new.

## Column identity

The `field` value is the public identity of a column. Persistence adapters store column width,
order, and visibility by field name so state can survive remounts, reloads, and partially matching
schemas.

```tsx
<DataTableColumn field="name" />
<DataTableColumn field="category" />
<DataTableColumn field="price" />
```

Use stable field names, even when the column label is localized or restyled.

```tsx
<DataTableColumn field="customerName">
  <DataTableColumnHeader>{locale === 'de' ? 'Kunde' : 'Customer'}</DataTableColumnHeader>
</DataTableColumn>
```

Changing the header label is a display change. Changing `field` is an identity change. The new field
will not match previously saved width, order, sticky, or visibility state.

## Row identity

Use `computeRowKey` when rows have stable IDs. It receives the row data, row index, and table
context, and should return the React key for that row.

```tsx
<DataTable computeRowKey={({ data }) => data.id} model={model}>
  {/* columns */}
</DataTable>
```

Without `computeRowKey`, row identity falls back to the virtual row position. That is acceptable for
static lists, but it is fragile when users sort, filter, group, prepend, append, or replace rows.
Stable row keys prevent unnecessary remounts when the same logical row moves to a different
position.

If grouped data mixes data rows and group rows, make sure both shapes return stable keys.

```tsx
<DataTable
  computeRowKey={({ data, index }) => ('id' in data ? data.id : `group:${data.label}:${index}`)}
  model={model}
>
  {/* columns */}
</DataTable>
```

## Context

Use the table `context` prop for state that table infrastructure needs but that does not belong in
each row: empty-state copy, row component styling flags, sticky wrapper settings, custom scroll
element state, or row-key prefixes. `context` is passed to table infrastructure callbacks and
component overrides, not to ordinary cell and header renderers.

```tsx
function EmptyPlaceholder({ context }) {
  return <div>{context.emptyMessage}</div>
}

;<DataTable
  EmptyPlaceholder={EmptyPlaceholder}
  computeRowKey={({ data, context }) => `${context.tenantId}:${data.id}`}
  context={{ emptyMessage: 'No archived products', tenantId: 'acme' }}
  model={model}
>
  {/* columns */}
</DataTable>
```

The same `context` value is passed to `computeRowKey`, `EmptyPlaceholder`, loading slots, row
component overrides, sticky wrapper overrides, and custom scroll elements.

Cell and header render functions do not receive `context`. Close over screen-local state directly
when a renderer needs it.
