Column and row identity affect persistence, virtualization, and remount behavior. Keep both stable
unless you intentionally want the table to treat something as new.

## Column identity

The `field` value is the public identity of a column. Persistence adapters store column width,
order, and visibility by field name so state can survive remounts and partially matching schemas.

```tsx
<DataTableColumn field="name" />
<DataTableColumn field="category" />
<DataTableColumn field="price" />
```

Use stable field names, even when the column label is localized or restyled.

```tsx
<DataTableColumn field="customerName">
  <DataTableColumnHeader>{locale === 'de' ? 'Kunde' : 'Customer'}</DataTableColumnHeader>
</DataTableColumn>
```

Changing the header label is a display change. Changing `field` is an identity change and may reset
saved width, order, sticky, and visibility state for that column.

## Row identity

Use `computeRowKey` when rows have stable IDs. It receives the row data, row index, and table
context, and should return the React key for that row.

```tsx
<DataTable computeRowKey={({ data }) => data.id} model={model}>
  {/* columns */}
</DataTable>
```

Stable row keys prevent unnecessary remounts when sorting, filtering, grouping, or replacing the
data array. If grouped data mixes data rows and group rows, make sure group rows also return stable
keys.

## Context

Use the table `context` prop for state that table infrastructure needs but that does not belong in
each row: empty-state copy, row component styling flags, sticky wrapper settings, custom scroll
element state, or row-key prefixes.

```tsx
function EmptyPlaceholder({ context }) {
  return <div>{context.emptyMessage}</div>
}

;<DataTable
  EmptyPlaceholder={EmptyPlaceholder}
  computeRowKey={({ data, context }) => `${context.tenantId}:${data.id}`}
  context={{ emptyMessage: 'No archived products', tenantId: 'acme' }}
  model={model}
>
  {/* columns */}
</DataTable>
```

The same `context` value is passed to `computeRowKey`, `EmptyPlaceholder`, loading slots, row
component overrides, sticky wrapper overrides, and custom scroll elements.

Cell and header render functions do not receive `context`. Close over screen-local state directly
when a renderer needs it.
