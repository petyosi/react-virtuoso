Cell and header components define renderers. The table calls those renderers when it needs to paint a
header cell or a body cell. This lets the table virtualize rows and columns while your JSX remains a
declarative description of the table structure.

The shadcn wrapper accepts plain header text for convenience:

```tsx
<DataTableColumnHeader>Product</DataTableColumnHeader>
```

It also accepts render functions when the header needs table state:

```tsx
<DataTableColumnHeader>{({ columnState }) => (columnState.sticky ? 'Pinned product' : 'Product')}</DataTableColumnHeader>
```

## Cell values and rows

Use `cellValue` when the cell only needs the selected field. Use `row.data` when formatting depends
on other fields.

```tsx live
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'
import { localModel } from '@virtuoso.dev/data-table'

const rows = [
  { id: 'SKU-001', name: 'Standing Desk', category: 'Office', price: 699, stock: 14 },
  { id: 'SKU-002', name: 'USB-C Dock', category: 'Peripherals', price: 229, stock: 42 },
  { id: 'SKU-003', name: 'Mechanical Keyboard', category: 'Peripherals', price: 169, stock: 28 },
]

const currency = new Intl.NumberFormat('en-US', {
  currency: 'USD',
  style: 'currency',
})

const model = localModel({ data: rows })

export default function App() {
  return (
    <DataTable className="rounded-xl" model={model} style={{ height: 280 }}>
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell className="font-medium">{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="price">
        <DataTableColumnHeader className="justify-end">Price</DataTableColumnHeader>
        <DataTableCell className="text-right tabular-nums">{({ cellValue }) => currency.format(Number(cellValue))}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="stock">
        <DataTableColumnHeader>Status</DataTableColumnHeader>
        <DataTableCell>
          {({ row }) => (
            <span className={row.data.stock < 20 ? 'font-medium text-amber-600' : 'text-foreground'}>
              {row.data.stock < 20 ? 'Low stock' : `${row.data.stock} in stock`}
            </span>
          )}
        </DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Render params

Cell renderers receive:

- `cellValue`, the value at `row.data[column.field]`
- `row`, the virtualized row item
- `column`, the column definition
- `columnKey`, the internal key for this column instance
- `columnState`, currently including sticky state
- `overlaidByScrollbar`, useful when a native scrollbar overlaps the cell edge

Header renderers receive the column-specific values: `column`, `columnKey`, `columnState`, and
`overlaidByScrollbar`.

```tsx
<DataTableColumnHeader>
  {({ column, columnState }) => <span className={columnState.sticky ? 'font-semibold' : undefined}>{column.field}</span>}
</DataTableColumnHeader>
```

## Extracting renderers

Use the exported render-param types when extracting renderers into named functions.

```tsx
import type { CellRenderParams, ColumnHeaderRenderParams } from '@virtuoso.dev/data-table'

function StockCell({ row }: CellRenderParams) {
  return <span>{row.data.stock < 20 ? 'Low stock' : 'Healthy'}</span>
}

function FieldHeader({ column }: ColumnHeaderRenderParams) {
  return <span>{column.field}</span>
}
```

Then pass them as render children:

```tsx
<DataTableColumn field="stock">
  <DataTableColumnHeader>{FieldHeader}</DataTableColumnHeader>
  <DataTableCell>{StockCell}</DataTableCell>
</DataTableColumn>
```

## Header slots

Header slots are separate definitions that attach UI around a column header, such as resize handles,
sort controls, or custom overlays. They are declared inside the header definition but rendered by
the table beside the header content.

```tsx
<DataTableColumnHeader>
  {() => 'Price'}
  <HeaderEnd>{() => <button type="button">Sort</button>}</HeaderEnd>
</DataTableColumnHeader>
```

The shadcn wrapper uses this mechanism for feature controls. See
[Adding Controls to Column Headers](/data-table/customization/header-slots/) for custom slot placement.
