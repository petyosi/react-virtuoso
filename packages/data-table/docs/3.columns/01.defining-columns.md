A column starts with `DataTableColumn`. Its `field` prop is both the field lookup for row data and
the public column key used by table features.

```tsx
<DataTableColumn field="name">
  <DataTableColumnHeader>Name</DataTableColumnHeader>
  <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
</DataTableColumn>
```

For a row like `{ name: 'Standing Desk' }`, the cell renderer receives `'Standing Desk'` as
`cellValue`. If you omit `DataTableCell`, the table renders that value as text. If you omit
`DataTableColumnHeader`, the table uses the field name as the header label.

## Minimal columns

Use explicit headers and cells when you need formatting. Leave them out when the field name and
stringified value are enough.

```tsx live
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'
import { localModel } from '@virtuoso.dev/data-table'

const rows = [
  { id: 'SKU-001', name: 'Standing Desk', category: 'Office', stock: 14 },
  { id: 'SKU-002', name: 'USB-C Dock', category: 'Peripherals', stock: 42 },
  { id: 'SKU-003', name: 'Mechanical Keyboard', category: 'Peripherals', stock: 28 },
]

const model = localModel({ data: rows })

export default function App() {
  return (
    <DataTable className="rounded-xl" model={model} style={{ height: 260 }}>
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell className="font-medium">{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="category" />
      <DataTableColumn field="stock">
        <DataTableColumnHeader className="justify-end">Stock</DataTableColumnHeader>
        <DataTableCell className="text-right tabular-nums">{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Declaration order

Columns render in the order they are declared. Feature state can change the runtime order later,
but the declaration order remains the stable baseline for persistence and resets.

```tsx
<DataTable model={model}>
  <DataTableColumn field="name" />
  <DataTableColumn field="category" />
  <DataTableColumn field="price" />
</DataTable>
```

## Visibility and sticky columns

The `visible` prop controls the initial declarative visibility. Runtime visibility changes are
handled by the column visibility feature.

```tsx
<DataTableColumn field="internalSku" visible={false} />
```

The `sticky` prop marks a column as initially sticky on the left or right. Use it for row labels,
IDs, or row actions that should remain visible while the user scrolls horizontally.

```tsx
<DataTableColumn field="name" sticky="left" />
<DataTableColumn field="actions" sticky="right" />
```

For interactive controls, see [Column Visibility](/data-table/features/column-visibility/) and
[Sticky Columns](/data-table/features/sticky-columns/).

## Generated columns

If the model data has dynamic fields, render columns from a stable field list. The field values
still become the column keys, so keep their order deterministic.

```tsx
const fields = ['name', 'category', 'stock']

;<DataTable model={model}>
  {fields.map((field) => (
    <DataTableColumn field={field} key={field}>
      <DataTableColumnHeader>{field}</DataTableColumnHeader>
      <DataTableCell>{({ cellValue }) => String(cellValue ?? '')}</DataTableCell>
    </DataTableColumn>
  ))}
</DataTable>
```

For schemas inferred from model data, see
[Columns for Dynamic Schemas](/data-table/columns/inferring-columns-from-model-data/).
