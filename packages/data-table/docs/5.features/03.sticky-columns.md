title: Sticky Columns
description: Pin high-value columns to the left or right side of a wide table.

---

Set `sticky="left"` or `sticky="right"` on a column when users need that column to stay visible
during horizontal scrolling.

Sticky columns work best for row identity, status, and action columns. Give sticky columns enough
width to remain readable, and avoid pinning so many columns that the scrollable area becomes too
narrow.

```tsx live wide
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'
import { localModel } from '@virtuoso.dev/data-table'

const rows = Array.from({ length: 40 }, (_, index) => ({
  id: `SKU-${String(index + 1).padStart(3, '0')}`,
  name: `Product ${index + 1}`,
  category: ['Office', 'Peripherals', 'Audio'][index % 3]!,
  status: ['In stock', 'Low stock', 'Backorder'][index % 3]!,
  owner: ['Avery', 'Morgan', 'Riley'][index % 3]!,
  region: ['US', 'EU', 'APAC'][index % 3]!,
  supplier: ['Northwind', 'Contoso', 'Fabrikam'][index % 3]!,
  warehouse: ['Denver', 'Rotterdam', 'Singapore'][index % 3]!,
  price: 49 + (index % 8) * 24,
  stock: (index * 7) % 64,
  leadTime: `${2 + (index % 5)} days`,
  margin: `${18 + (index % 6) * 3}%`,
  updated: `2026-05-${String(1 + (index % 17)).padStart(2, '0')}`,
}))

const model = localModel({ data: rows })
const scrollColumns = [
  ['category', 'Category'],
  ['owner', 'Owner'],
  ['region', 'Region'],
  ['supplier', 'Supplier'],
  ['warehouse', 'Warehouse'],
  ['price', 'Price'],
  ['stock', 'Stock'],
  ['leadTime', 'Lead time'],
  ['margin', 'Margin'],
  ['updated', 'Updated'],
] as const

export default function App() {
  return (
    <DataTable className="rounded-xl" computeRowKey={({ data }) => data.id} model={model} style={{ height: 340 }}>
      <DataTableColumn field="name" sticky="left">
        <DataTableColumnHeader className="min-w-[180px]">Product</DataTableColumnHeader>
        <DataTableCell className="min-w-[180px] font-medium">{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
      {scrollColumns.map(([field, label]) => (
        <DataTableColumn key={field} field={field}>
          <DataTableColumnHeader className="min-w-[140px]">{label}</DataTableColumnHeader>
          <DataTableCell className="min-w-[140px]">{({ cellValue }) => String(cellValue)}</DataTableCell>
        </DataTableColumn>
      ))}
      <DataTableColumn field="status" sticky="right">
        <DataTableColumnHeader className="min-w-[140px]">Status</DataTableColumnHeader>
        <DataTableCell className="min-w-[140px]">{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

```tsx
<DataTableColumn field="name" sticky="left">
  <DataTableColumnHeader>Product</DataTableColumnHeader>
  <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
</DataTableColumn>

<DataTableColumn field="region" sticky="right">
  <DataTableColumnHeader>Region</DataTableColumnHeader>
  <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
</DataTableColumn>
```

The `min-w-*` classes in the example make the horizontal scroll area predictable. Without stable
minimum widths, sticky columns can become cramped or overlap visually with scrollable content.

You can also change sticky state from outside the column declaration with `setColumnSticky$` and the
table engine hooks. See the [Sticky Columns example](/data-table/examples/sticky-columns/) for a
runnable external-toggle pattern, and [Controlling the Table From Surrounding UI](/data-table/features/table-instance-control/)
for the underlying engine API.
