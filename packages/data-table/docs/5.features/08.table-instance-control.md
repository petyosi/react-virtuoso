Table instance control is the public contract for coordinating a table with surrounding UI. Use it
when a toolbar, side panel, or sibling widget needs to read table state or publish actions without
turning every table detail into React state.

The table still owns virtualization, measurement, and feature state. External UI receives a stable
handle to the table engine, reads supported state cells, and publishes supported actions.

## Engine refs

```tsx live
import { Button } from '@/components/ui/button'
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'
import { localModel, scrollLocation$, scrollToRow$, useEngineRef, useRemoteCellValue, useRemotePublisher } from '@virtuoso.dev/data-table'

const rows = Array.from({ length: 120 }, (_, index) => ({
  id: `SKU-${String(index + 1).padStart(3, '0')}`,
  name: `Product ${index + 1}`,
  category: ['Office', 'Peripherals', 'Audio'][index % 3]!,
}))

const model = localModel({ data: rows })

export default function App() {
  const engineRef = useEngineRef()
  const scrollLocation = useRemoteCellValue(scrollLocation$, engineRef)
  const scrollToRow = useRemotePublisher(scrollToRow$, engineRef)

  return (
    <div className="space-y-4">
      <div className="flex flex-wrap gap-2">
        <Button onClick={() => scrollToRow({ index: 0, behavior: 'smooth' })} variant="outline">
          Top
        </Button>
        <Button onClick={() => scrollToRow({ index: 80, align: 'center', behavior: 'smooth' })} variant="outline">
          Center row 81
        </Button>
      </div>

      <div className="rounded-lg border bg-muted/40 px-3 py-2 text-sm text-muted-foreground">
        {scrollLocation === undefined ? 'Waiting for scroll state...' : `Visible height: ${Math.round(scrollLocation.visibleListHeight)}px`}
      </div>

      <DataTable className="rounded-xl" model={model} engineRef={engineRef} style={{ height: 360 }}>
        <DataTableColumn field="name">
          <DataTableColumnHeader>Product</DataTableColumnHeader>
          <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
        </DataTableColumn>
        <DataTableColumn field="category">
          <DataTableColumnHeader>Category</DataTableColumnHeader>
          <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
        </DataTableColumn>
      </DataTable>
    </div>
  )
}
```

Use `engineRef` when the controls are rendered near one specific table. This is the most direct
option for page-local toolbars and dashboards.

## Engine IDs

Use `engineId` when the reader or publisher lives elsewhere in the React tree, such as a page-level
toolbar or layout shell.

```tsx
import { Button } from '@/components/ui/button'
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'
import { scrollToRow$, useRemotePublisher } from '@virtuoso.dev/data-table'

const TABLE_ENGINE_ID = 'inventory-table'

function PageToolbar() {
  const scrollToRow = useRemotePublisher(scrollToRow$, TABLE_ENGINE_ID)

  return (
    <Button onClick={() => scrollToRow({ index: 0, behavior: 'smooth' })} variant="outline">
      Top
    </Button>
  )
}

function InventoryTable({ rows }) {
  return (
    <DataTable model={model} engineId={TABLE_ENGINE_ID} style={{ height: 420 }}>
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Common state and actions

- `columns$`, `visibleColumns$`, `columnWidths$`, `scrollLocation$`, `scrollerElement$`, and
  `itemHeight$` expose table state.
- `scrollToRow$`, `scrollIntoView$`, `cancelSmoothScroll$`, and `setColumnSticky$` publish core table actions.
- Feature modules add their own actions, such as `resizeColumn$`, `reorderColumns$`, and
  `setColumnVisibility$`.
- Child components inside the table can use `useVirtuosoLocation()`, `useCurrentlyRenderedData()`,
  and `useVirtuosoLoadingState()` for local reads.

Treat these streams and cells as the supported table instance API. Prefer the state cells over DOM
inspection; `scrollerElement$` and `itemHeight$` are available for integration cases that need the
actual scroller node or a measured row height.

## Scroll lifecycle props

Use props for one-way table lifecycle notifications and first-render positioning:

- `initialLocation` scrolls to a row when the table first renders with data.
- `onScroll` receives the current `ListScrollLocation`.
- `onRenderedDataChange` receives the currently rendered data rows, excluding group rows.

```tsx
<DataTable
  model={model}
  initialLocation={{ index: 80, align: 'center' }}
  onRenderedDataChange={(renderedRows) => {
    console.log(renderedRows.map((row) => row.id))
  }}
  onScroll={(location) => {
    console.log(location.scrollTop)
  }}
>
  {/* columns */}
</DataTable>
```

Use remote cells and actions when surrounding UI needs to read or change state. Use these props when
the table only needs to notify the owner.

## Overscan controls

Vertical virtualization is automatic. Tune overscan only when a specific table needs extra rendered
content:

- `increaseViewportBy` renders additional vertical pixels before and after the viewport.
- `columnOverscanCount` renders additional columns on each side of the horizontal viewport.

Start with the defaults. Increase these values for heavier cells, images that should warm up before
scrolling, or wide tables that need smoother horizontal movement.
