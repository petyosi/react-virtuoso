A "scroll to row" button in the page toolbar, a column-visibility menu beside the table, a status badge that reflects the current loading phase — UI outside the table often needs to read its state or trigger its actions. That communication happens through the table's internal reactive runtime, not by lifting state up into React props.

The connection point is a stable handle — either an `engineRef` returned by `useEngineRef()`, or a string `engineId` shared between the table and the external UI. State cells (`columns$`, `scrollLocation$`, …) are the public surface for reading; action streams (`scrollToRow$`, `setColumnVisibility$`, …) are the surface for publishing.

## Engine refs

```tsx live
import { Button } from '@/components/ui/button'
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'
import { localModel, scrollLocation$, scrollToRow$, useEngineRef, useRemoteCellValue, useRemotePublisher } from '@virtuoso.dev/data-table'

const rows = Array.from({ length: 120 }, (_, index) => ({
  id: `SKU-${String(index + 1).padStart(3, '0')}`,
  name: `Product ${index + 1}`,
  category: ['Office', 'Peripherals', 'Audio'][index % 3]!,
}))

const model = localModel({ data: rows })

export default function App() {
  const engineRef = useEngineRef()
  const scrollLocation = useRemoteCellValue(scrollLocation$, engineRef)
  const scrollToRow = useRemotePublisher(scrollToRow$, engineRef)

  return (
    <div className="space-y-4">
      <div className="flex flex-wrap gap-2">
        <Button onClick={() => scrollToRow({ index: 0, behavior: 'smooth' })} variant="outline">
          Top
        </Button>
        <Button onClick={() => scrollToRow({ index: 80, align: 'center', behavior: 'smooth' })} variant="outline">
          Center row 81
        </Button>
      </div>

      <div className="rounded-lg border bg-muted/40 px-3 py-2 text-sm text-muted-foreground">
        {scrollLocation === undefined ? 'Waiting for scroll state...' : `Visible height: ${Math.round(scrollLocation.visibleListHeight)}px`}
      </div>

      <DataTable className="rounded-xl" model={model} engineRef={engineRef} style={{ height: 360 }}>
        <DataTableColumn field="name">
          <DataTableColumnHeader>Product</DataTableColumnHeader>
          <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
        </DataTableColumn>
        <DataTableColumn field="category">
          <DataTableColumnHeader>Category</DataTableColumnHeader>
          <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
        </DataTableColumn>
      </DataTable>
    </div>
  )
}
```

`engineRef` is the direct option for page-local toolbars and dashboards rendered next to a specific table.

## Engine IDs

`engineId` lets a reader or publisher elsewhere in the tree — a page-level toolbar, layout shell — connect to the table by string identifier.

```tsx
import { Button } from '@/components/ui/button'
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'
import { scrollToRow$, useRemotePublisher } from '@virtuoso.dev/data-table'

const TABLE_ENGINE_ID = 'inventory-table'

function PageToolbar() {
  const scrollToRow = useRemotePublisher(scrollToRow$, TABLE_ENGINE_ID)

  return (
    <Button onClick={() => scrollToRow({ index: 0, behavior: 'smooth' })} variant="outline">
      Top
    </Button>
  )
}

function InventoryTable({ rows }) {
  return (
    <DataTable model={model} engineId={TABLE_ENGINE_ID} style={{ height: 420 }}>
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Common state and actions

State cells:

- `columns$`, `visibleColumns$`, `columnWidths$` — column shape and layout
- `scrollLocation$`, `scrollerElement$`, `itemHeight$` — scroll state and DOM references

Core actions:

- `scrollToRow$`, `scrollIntoView$`, `cancelSmoothScroll$` — scroll commands
- `setColumnSticky$` — pin a column at runtime

Feature modules contribute their own: `resizeColumn$`, `reorderColumns$`, `setColumnVisibility$`.

Child components inside the table use `useVirtuosoLocation()`, `useCurrentlyRenderedData()`, and `useVirtuosoLoadingState()` for local reads.

Reach for state cells before DOM inspection. `scrollerElement$` and `itemHeight$` exist for integrations that need the raw scroller node or a measured row height.

## Scroll lifecycle props

One-way notifications and first-render positioning live on the component, not the engine:

- `initialLocation` — scrolls to a row when the table first renders with data
- `onScroll` — receives the current `ListScrollLocation`
- `onRenderedDataChange` — receives the rendered data rows, excluding group rows

```tsx
<DataTable
  model={model}
  initialLocation={{ index: 80, align: 'center' }}
  onRenderedDataChange={(renderedRows) => {
    console.log(renderedRows.map((row) => row.id))
  }}
  onScroll={(location) => {
    console.log(location.scrollTop)
  }}
>
  {/* columns */}
</DataTable>
```

Props are right when the parent just needs to observe; remote cells and actions are right when external UI needs to read or change table state.

## Overscan controls

Virtualization is automatic. Tune overscan only when a specific table needs extra rendered content:

- `increaseViewportBy` — extra vertical pixels rendered before and after the viewport
- `columnOverscanCount` — extra columns rendered on each side of the horizontal viewport

Defaults are good. Raise overscan for heavier cells, images that need to warm up before scrolling, or wide tables that feel jumpy during horizontal movement.
