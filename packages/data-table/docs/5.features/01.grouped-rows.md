Group headers and data rows share one flat array. `groups: [{ index, level }]` marks which entries are headers; the table renders headers through `GroupHeaderCell`.

```ts
const tableData = {
  data: [
    { label: 'Office (2)' },
    { id: 'SKU-001', name: 'Standing Desk', category: 'Office' },
    { id: 'SKU-002', name: 'Desk Lamp', category: 'Office' },
  ],
  groups: [{ index: 0, level: 0 }],
}
```

`index` points into `data`. `level` drives nested-group styling.

Pass the shape to `localModel({ data, groups })`. `GroupHeaderCell` receives `row.data` for the entry at that index:

```tsx
const model = localModel(tableData)

;<DataTable model={model}>
  <GroupHeaderCell>{({ row }) => row.data.label}</GroupHeaderCell>
  <DataTableColumn field="name">
    <DataTableColumnHeader>Product</DataTableColumnHeader>
    <DataTableCell>{({ cellValue }) => String(cellValue ?? '')}</DataTableCell>
  </DataTableColumn>
</DataTable>
```

```tsx live
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader, GroupHeaderCell } from '@/components/ui/data-table'
import { localModel } from '@virtuoso.dev/data-table'

const rows = [
  { label: 'Office' },
  { id: 'SKU-001', name: 'Standing Desk', category: 'Office', stock: 14 },
  { id: 'SKU-002', name: 'Desk Lamp', category: 'Office', stock: 31 },
  { label: 'Peripherals' },
  { id: 'SKU-003', name: 'USB-C Dock', category: 'Peripherals', stock: 42 },
  { id: 'SKU-004', name: 'Mechanical Keyboard', category: 'Peripherals', stock: 28 },
]

const groups = [
  { index: 0, level: 0 },
  { index: 3, level: 0 },
]

const model = localModel({ data: rows, groups })

export default function App() {
  return (
    <DataTable
      className="rounded-xl"
      computeRowKey={({ data, index }) => ('id' in data ? data.id : `group-${index}`)}
      model={model}
      style={{ height: 320 }}
    >
      <GroupHeaderCell>
        {({ row }) => <div className="bg-muted/60 px-3 py-2 text-sm font-medium">{String(row.data.label)}</div>}
      </GroupHeaderCell>
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell className="font-medium">{({ cellValue }) => String(cellValue ?? '')}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="stock">
        <DataTableColumnHeader className="justify-end">Stock</DataTableColumnHeader>
        <DataTableCell className="text-right tabular-nums">{({ cellValue }) => String(cellValue ?? '')}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Local grouped data

`localModel()` accepts groups directly, or a pipeline stage can produce them:

```tsx
const model = localModel({
  data: products,
  pipeline: ['filter', 'group'],
  actions: {
    group: {
      stage: 'group',
      handler: ({ data, payload }) => groupProducts(data, payload as 'category' | 'status'),
    },
  },
})
```

A grouping handler returns `{ data, groups }`; an ungrouped handler returns a flat array. Outside the pipeline, `model.setData(nextRows, nextGroups)` updates rows and markers together.

## Remote grouped data

`remoteModel()` forwards group markers from the fetch result:

```ts
async function fetchProducts(params: FetchParams<Query>) {
  const grouped = await fetchGroupedRows(params)

  return {
    rows: grouped.data,
    groups: grouped.groups,
    totalCount: grouped.data.length,
  }
}
```

In offset mode, marker indexes are positions in the full flattened result. In append mode, return markers for the row stream after each append. Group rows need a shape your `GroupHeaderCell` can branch on.

## Rules of thumb

- Group labels are virtualized rows — keep them lightweight.
- `level` is for visual hierarchy. It doesn't change row behavior.
- The table doesn't compute groups; group your data before passing it in.
