Grouped rows are a data feature. A grouped table flattens group labels and normal rows into one
virtualized row stream, then marks the group rows with `groups`.

```ts
const tableData = {
  data: [
    { label: 'Office (2)' },
    { id: 'SKU-001', name: 'Standing Desk', category: 'Office' },
    { id: 'SKU-002', name: 'Desk Lamp', category: 'Office' },
  ],
  groups: [{ index: 0, level: 0 }],
}
```

`index` points to the group row inside `data`. `level` lets you style nested groups differently.

Pass grouped rows through `localModel({ data, groups })` and render group rows with
`GroupHeaderCell`. The `row.data` value is the object at the corresponding group index in the
flattened `data` array.

```tsx
const model = localModel(tableData)

;<DataTable model={model}>
  <GroupHeaderCell>{({ row }) => row.data.label}</GroupHeaderCell>
  <DataTableColumn field="name">
    <DataTableColumnHeader>Product</DataTableColumnHeader>
    <DataTableCell>{({ cellValue }) => String(cellValue ?? '')}</DataTableCell>
  </DataTableColumn>
</DataTable>
```

```tsx live
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader, GroupHeaderCell } from '@/components/ui/data-table'
import { localModel } from '@virtuoso.dev/data-table'

const rows = [
  { label: 'Office' },
  { id: 'SKU-001', name: 'Standing Desk', category: 'Office', stock: 14 },
  { id: 'SKU-002', name: 'Desk Lamp', category: 'Office', stock: 31 },
  { label: 'Peripherals' },
  { id: 'SKU-003', name: 'USB-C Dock', category: 'Peripherals', stock: 42 },
  { id: 'SKU-004', name: 'Mechanical Keyboard', category: 'Peripherals', stock: 28 },
]

const groups = [
  { index: 0, level: 0 },
  { index: 3, level: 0 },
]

const model = localModel({ data: rows, groups })

export default function App() {
  return (
    <DataTable
      className="rounded-xl"
      computeRowKey={({ data, index }) => ('id' in data ? data.id : `group-${index}`)}
      model={model}
      style={{ height: 320 }}
    >
      <GroupHeaderCell>
        {({ row }) => <div className="bg-muted/60 px-3 py-2 text-sm font-medium">{String(row.data.label)}</div>}
      </GroupHeaderCell>
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell className="font-medium">{({ cellValue }) => String(cellValue ?? '')}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="stock">
        <DataTableColumnHeader className="justify-end">Stock</DataTableColumnHeader>
        <DataTableCell className="text-right tabular-nums">{({ cellValue }) => String(cellValue ?? '')}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Local grouped data

`localModel()` can start with groups or return groups from a pipeline stage.

```tsx
const model = localModel({
  data: products,
  pipeline: ['filter', 'group'],
  actions: {
    group: {
      stage: 'group',
      handler: ({ data, payload }) => groupProducts(data, payload as 'category' | 'status'),
    },
  },
})
```

Return a flat array for ungrouped results and `{ data, groups }` for grouped results. If the source
rows change outside the pipeline, `model.setData(nextRows, nextGroups)` updates both rows and group
markers.

## Remote grouped data

`remoteModel()` preserves group metadata returned from the fetch function.

```ts
async function fetchProducts(params: FetchParams<Query>) {
  const grouped = await fetchGroupedRows(params)

  return {
    rows: grouped.data,
    groups: grouped.groups,
    totalCount: grouped.data.length,
  }
}
```

For offset mode, group indexes are indexes in the total flattened result. For append mode, return
markers for the accumulated row stream after the append. Group rows should use a shape your
`GroupHeaderCell` can recognize.

## Patterns

- Keep group labels lightweight because they are virtualized rows.
- Use `level` for visual hierarchy, not for data ownership.
- Group data before it reaches the table. The table renders the supplied grouped shape; it does not
  compute groups for you.

See [Group Rows by Region and Category](/data-table/examples/grouped-data/) for a complete local model table,
and [Build an In-memory Data Pipeline](/data-table/examples/local-filter-sort-group/) for grouping inside a
local model pipeline.
