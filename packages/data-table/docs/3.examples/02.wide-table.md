This recipe is useful for metrics-heavy tables where the field count is just as important as the row count.

## APIs that matter

- `columnOverscanCount` for smoother horizontal scrolling
- generated `DataTableColumn` definitions for repeated structures

```tsx live wide file=App.tsx
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'

import { columns, rows } from './data'

export default function App() {
  return (
    <DataTable className="rounded-xl" columnOverscanCount={2} data={{ data: rows, groups: [] }} style={{ height: 400 }}>
      {columns.map((column) => (
        <DataTableColumn field={column} key={column}>
          <DataTableColumnHeader className="min-w-[130px]">{column}</DataTableColumnHeader>
          <DataTableCell className="min-w-[130px]">{({ row }) => String(row.data[column])}</DataTableCell>
        </DataTableColumn>
      ))}
    </DataTable>
  )
}
```

```ts live file=data.ts
export const columns = ['sku', ...Array.from({ length: 24 }, (_, index) => `metric${index + 1}`)]

export const rows = Array.from({ length: 120 }, (_, rowIndex) => {
  const row: Record<string, string> = {
    sku: `SKU-${String(rowIndex + 1).padStart(3, '0')}`,
  }

  for (const column of columns.slice(1)) {
    row[column] = `R${rowIndex + 1}-${column}`
  }

  return row
})
```

## Tradeoffs

This keeps authoring compact, but generated fields usually deserve a separate schema or column metadata object once you add sorting or custom formatting.
