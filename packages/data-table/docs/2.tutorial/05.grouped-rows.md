Grouped tables flatten both the group labels and the real rows into one virtualized stream. The `groups` array marks which indexes are group rows and at which nesting level they live.

## Nested category view

```tsx live wide file=App.tsx
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader, GroupHeaderCell } from '@/components/ui/data-table'

import { groupedProducts } from './data'

const currency = new Intl.NumberFormat('en-US', {
  currency: 'USD',
  style: 'currency',
})

export default function App() {
  return (
    <DataTable className="rounded-xl" data={groupedProducts} style={{ height: 420 }}>
      <GroupHeaderCell>
        {({ level, row }) => (
          <div className={level === 0 ? 'bg-muted px-3 py-2 font-semibold' : 'bg-muted/50 px-6 py-2 font-medium'}>
            {(row.data as { label: string }).label}
          </div>
        )}
      </GroupHeaderCell>

      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell className="font-medium">{({ row }) => row.data.name}</DataTableCell>
      </DataTableColumn>

      <DataTableColumn field="status">
        <DataTableColumnHeader>Status</DataTableColumnHeader>
        <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>

      <DataTableColumn field="revenue">
        <DataTableColumnHeader className="justify-end">Revenue</DataTableColumnHeader>
        <DataTableCell className="text-right tabular-nums">{({ cellValue }) => currency.format(Number(cellValue))}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

```ts live file=data.ts
export interface Product {
  id: string
  name: string
  category: 'Audio' | 'Office' | 'Peripherals'
  revenue: number
  region: 'US' | 'EU'
  status: 'In stock' | 'Low stock' | 'Backorder'
}

interface GroupLabel {
  label: string
}

const categories: Product['category'][] = ['Office', 'Peripherals', 'Audio']
const regions: Product['region'][] = ['US', 'EU']

const products: Product[] = Array.from({ length: 90 }, (_, index) => {
  const stock = (index * 5) % 32

  return {
    id: `SKU-${String(index + 1).padStart(3, '0')}`,
    name: `${categories[index % categories.length]} Item ${index + 1}`,
    category: categories[index % categories.length]!,
    revenue: 1_100 + index * 85,
    region: regions[index % regions.length]!,
    status: stock < 6 ? 'Backorder' : stock < 12 ? 'Low stock' : 'In stock',
  }
})

export const groupedProducts: {
  data: (GroupLabel | Product)[]
  groups: { index: number; level: number }[]
} = (() => {
  const data: (GroupLabel | Product)[] = []
  const groups: { index: number; level: number }[] = []

  for (const region of regions) {
    groups.push({ index: data.length, level: 0 })
    data.push({ label: `Region: ${region}` })

    for (const category of categories) {
      const rows = products.filter((product) => product.region === region && product.category === category)
      groups.push({ index: data.length, level: 1 })
      data.push({ label: `${category} (${rows.length})` })
      data.push(...rows)
    }
  }

  return { data, groups }
})()
```

## Notes

- Level `0` groups are useful for broad sections like region or department.
- Level `1` and deeper groups work well for category, vendor, or lifecycle status buckets.
- The group rows participate in virtualization just like normal rows, so large grouped datasets stay cheap to render.
