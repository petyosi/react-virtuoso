The recommended starting point is the shadcn registry wrapper. It installs editable app code under
`@/components/ui/data-table` while keeping the headless engine in `@virtuoso.dev/data-table`.

The wrapper owns:

- default row, header, sticky-column, empty, and loading presentation
- Tailwind classes for cells, headers, and group headers
- ergonomic names such as `DataTable`, `DataTableColumn`, `DataTableColumnHeader`, and
  `DataTableCell`
- re-exports for the headless components and hooks commonly used with the wrapper

The package owns:

- virtualization, measurement, and scrolling behavior
- data model binding through `data`, `localSource()`, and `remoteSource()`
- reactive table state and action streams
- opt-in feature models such as resizing, reordering, visibility, and persistence

Because the wrapper is copied into your app, customize it like application code. Keep behavior imports
pointing at the headless package so your local styling changes do not fork table mechanics.

Registry feature components follow the same split:

- `data-table-reorder-grip`, `data-table-reorder-drop-zone`, and
  `data-table-draggable-group-header` provide reorder UI
- `data-table-resize-handle` provides resize UI

Those components are header-slot UI. Their model streams still come from the corresponding package
subpath.
