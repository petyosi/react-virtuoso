Scrolling and virtualization are built in. Only customize the scroll container when the table must
share a viewport with surrounding page chrome or a custom scroll component.

## Window scrolling

Use `useWindowScroll` when the document is the scroll container.

```tsx
<DataTable data={{ data: rows, groups: [] }} useWindowScroll>
  {/* columns */}
</DataTable>
```

In window-scroll mode the table does not own an internal scrolling element. The document viewport
drives vertical scroll state, so the table's height comes from its rendered content and the page
layout around it. Keep page chrome stable while the table is visible, and test sticky headers with
your actual header/nav stack because sticky offsets now interact with the document rather than a
contained table scroller.

## Existing parent element

Use `customScrollParent` when an existing element owns scrolling.

```tsx
const [scrollParent, setScrollParent] = React.useState<HTMLDivElement | null>(null)

return (
  <div ref={setScrollParent} className="h-[480px] overflow-auto">
    <Toolbar />
    <DataTable customScrollParent={scrollParent} data={{ data: rows, groups: [] }}>
      {/* columns */}
    </DataTable>
  </div>
)
```

## Custom scroll element

Use `ScrollElement` when you need to replace the table's default scroll element component.

```tsx
const ScrollElement = React.forwardRef<HTMLDivElement, React.HTMLProps<HTMLDivElement>>(function ScrollElement(props, ref) {
  return <div ref={ref} className="custom-scrollbar overflow-auto" {...props} />
})

<DataTable ScrollElement={ScrollElement} data={{ data: rows, groups: [] }} />
```

Choose only one scroll mode for a table. If you use `customScrollParent`, make sure that element has a
real height and is the element that actually scrolls.
