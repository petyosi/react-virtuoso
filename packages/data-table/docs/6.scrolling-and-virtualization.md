The table virtualizes both rows and columns, but you still control how aggressively it renders around the viewport and where it starts.

## Targeted example

```tsx live
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'

const rows = Array.from({ length: 200 }, (_, index) => ({
  id: `SKU-${String(index + 1).padStart(3, '0')}`,
  name: `Product ${index + 1}`,
  region: ['US', 'EU', 'APAC'][index % 3]!,
  status: ['In stock', 'Low stock', 'Backorder'][index % 3]!,
}))

export default function App() {
  return (
    <DataTable
      className="rounded-xl"
      columnOverscanCount={1}
      data={{ data: rows, groups: [] }}
      increaseViewportBy={120}
      initialLocation={{ index: 60, align: 'center' }}
      style={{ height: 360 }}
    >
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
        <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="region">
        <DataTableColumnHeader>Region</DataTableColumnHeader>
        <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
      <DataTableColumn field="status">
        <DataTableColumnHeader>Status</DataTableColumnHeader>
        <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Patterns

- `initialLocation` avoids work at the top of the list when you already know the useful starting row.
- `increaseViewportBy` helps when row content is visually heavy.
- `columnOverscanCount` is the horizontal equivalent for wide tables.
