Use `localModel()` when the row array is already in the browser. The source can be a static array,
React state pushed into the model with `setData`, or a client-side collection updated by user
actions.

```typescript
const model = localModel({ data: products })
```

Pass the model to the table:

```tsx
<DataTable model={model} style={{ height: 360 }}>
  {/* columns */}
</DataTable>
```

## Why pipelines exist

Filtering, sorting, and grouping usually stack on top of each other. A search box should filter the
source rows, a sort control should sort the filtered rows, and a grouping control should group the
sorted rows. Pipelines make that order explicit and cache each stage.

```typescript
const model = localModel({
  data: products,
  pipeline: ['filter', 'sort'],
  actions: {
    filter: {
      stage: 'filter',
      handler: ({ data, payload }) => data.filter((row) => row.category === payload),
    },
    sort: {
      stage: 'sort',
      handler: ({ data }) => data.toSorted((a, b) => a.name.localeCompare(b.name)),
    },
  },
})
```

When `filter` changes, the filter stage and the later sort stage rerun. When only `sort` changes,
the cached filter result is reused. That matters for larger local tables where filtering, grouping,
or tree construction is not free.

## Source mutators

Use a source mutator when an action changes the backing rows rather than just deriving a view. A
local action without a `stage` is a source mutator. Source mutators invalidate the full pipeline
because every derived stage depends on the updated source.

```typescript
const model = localModel({
  data: products,
  actions: {
    renameProduct: {
      handler: ({ source, payload }) => source.map((row) => (row.id === payload.id ? { ...row, name: payload.name } : row)),
    },
  },
})

model.send({ action: 'renameProduct', payload: { id: 'SKU-001', name: 'Desk Pro' } })
```

`model.send()` is the public way to publish model actions from buttons, inputs, effects, or table
controls. The action name selects the handler; the payload is application data that your handler
interprets.

## React state updates

Keep the model stable. When rows arrive from React state, a websocket, or a parent workflow, push the
next row set into the model instead of recreating the model.

```tsx
const model = React.useMemo(() => localModel<Product>({ data: initialProducts }), [])

React.useEffect(() => {
  model.setData?.(products)
}, [model, products])
```

Use `model.setData(nextRows, nextGroups)` for grouped rows.

If local rows can have different fields over time, keep the model stable and regenerate the column
declarations from a stable field list. See
[Columns for Dynamic Schemas](/data-table/columns/inferring-columns-from-model-data/).

## Group metadata

Groups are not nested objects. They are normal rows inside the flattened data array, plus markers
that tell the table which indexes are group headers.

```typescript
const rows = [
  { label: 'Office' },
  { id: 'SKU-001', name: 'Standing Desk', category: 'Office' },
  { id: 'SKU-002', name: 'Desk Lamp', category: 'Office' },
  { label: 'Peripherals' },
  { id: 'SKU-003', name: 'USB-C Dock', category: 'Peripherals' },
]

const groups = [
  { index: 0, level: 0 },
  { index: 3, level: 0 },
]

const model = localModel({ data: rows, groups })
```

`index` points at the group row in `data`. `level` is for visual hierarchy, such as top-level region
groups and nested category groups. The table uses the markers to render `GroupHeaderCell`, measure
group rows separately, and keep sticky group headers aligned.

## Action persistence

Persistence is useful when table controls should survive reloads or navigation: search terms, filter
values, sort choices, grouping mode, and other action payloads. The table persistence adapter stores
only the action state, not the full row array.

```typescript
const model = localModel({
  data: products,
  pipeline: ['filter'],
  actions: {
    filter: {
      stage: 'filter',
      persistence: true,
      handler: ({ data, payload }) => (payload ? data.filter((row) => row.category === payload) : data),
    },
  },
})
```

Use an object when the persisted shape needs a custom key or empty-state handling.

```typescript
const model = localModel({
  data: products,
  pipeline: ['filter'],
  actions: {
    filter: {
      stage: 'filter',
      persistence: {
        key: 'categoryFilter',
        isEmpty: (value) => !value,
      },
      handler: ({ data, payload }) => (payload ? data.filter((row) => row.category === payload) : data),
    },
  },
})
```

Pair this with `modelStatePersistenceAdapter()` on the [State Persistence](/data-table/features/state-persistence/)
page when you want the model action state saved with the rest of the table state.

`isEmpty` controls whether a payload should be omitted from saved state. Use it for values such as
empty strings, `"all"` filters, or `null` selections that mean "no active filter" in your UI.
