The table does not infer columns automatically. Columns are React declarations so you can control
labels, formatting, visibility, ordering, and interactive header behavior. When the row schema is
dynamic, infer the field list in your app and map it to `DataTableColumn` declarations.

## Direct data

For plain arrays, derive fields from a representative set of rows and keep the order stable.

```tsx
const fields = React.useMemo(() => {
  const seen = new Set<string>()
  const next: string[] = []

  for (const row of rows) {
    for (const field of Object.keys(row)) {
      if (!seen.has(field)) {
        seen.add(field)
        next.push(field)
      }
    }
  }

  return next
}, [rows])
```

Render those fields as column declarations:

```tsx
{
  fields.map((field) => (
    <DataTableColumn key={field} field={field}>
      <DataTableColumnHeader>{field}</DataTableColumnHeader>
      <DataTableCell>{({ cellValue }) => String(cellValue ?? '')}</DataTableCell>
    </DataTableColumn>
  ))
}
```

## Model data

For `localSource()` and `remoteSource()`, the model result is the best discovery point because it
already reflects filtering, grouping, placeholders, and schema changes.

```tsx
function fieldsFromResult(result: DataResult<Record<string, unknown>>) {
  const fields: string[] = []
  const seen = new Set<string>()

  for (const row of result.data) {
    if (!row) {
      continue
    }

    for (const field of Object.keys(row)) {
      if (!seen.has(field)) {
        seen.add(field)
        fields.push(field)
      }
    }
  }

  return fields
}

function useDiscoveredFields(model: DataModelHandle<Record<string, unknown>>) {
  const [fields, setFields] = React.useState<string[]>([])

  React.useEffect(() => {
    return model.subscribe((message) => {
      if (message.type === 'result') {
        setFields(fieldsFromResult(message.payload as DataResult<Record<string, unknown>>))
      }
    })
  }, [model])

  return fields
}
```

## Practical rules

- Prefer an explicit schema from your backend when one exists. It gives you stable labels, order, and
  type hints before the first row resolves.
- Filter out technical fields such as internal IDs when they should not become visible columns.
- Keep `field` stable across datasets so column persistence can restore matching width, order, and
  visibility state.
- If rows can include group labels, skip group-only fields or provide a `GroupHeaderCell` that handles
  them separately.
