A metrics-heavy table with twenty-five generated columns. `columnOverscanCount` smooths horizontal scrolling; the column declarations are mapped from a stable field list.

## APIs used

- `columnOverscanCount` — extra columns rendered beside the horizontal viewport
- Generated `DataTableColumn` declarations — one per entry in a stable field list

```tsx live wide file=App.tsx
import { DataTable, DataTableCell, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'

import { columns, rows } from './data'
import { localModel } from '@virtuoso.dev/data-table'

const model = localModel({ data: rows })

export default function App() {
  return (
    <DataTable className="rounded-xl" columnOverscanCount={2} model={model} style={{ height: 400 }}>
      {columns.map((column) => (
        <DataTableColumn field={column} key={column}>
          <DataTableColumnHeader className="min-w-[130px]">{column}</DataTableColumnHeader>
          <DataTableCell>{({ row }) => String(row.data[column])}</DataTableCell>
        </DataTableColumn>
      ))}
    </DataTable>
  )
}
```

```ts live file=data.ts
export const columns = ['sku', ...Array.from({ length: 24 }, (_, index) => `metric${index + 1}`)]

export const rows = Array.from({ length: 120 }, (_, rowIndex) => {
  const row: Record<string, string> = {
    sku: `SKU-${String(rowIndex + 1).padStart(3, '0')}`,
  }

  for (const column of columns.slice(1)) {
    row[column] = `R${rowIndex + 1}-${column}`
  }

  return row
})
```

## When to outgrow this shape

- You add per-column sorting, formatting, or alignment — move to a column metadata object (label, align, type) and map that.
- The backend defines the schema — see [Generating Columns at Runtime](/data-table/columns/runtime-columns/).
