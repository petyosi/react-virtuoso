The `context` prop is useful when supporting components need extra information that does not belong on every row.

## Targeted example

```tsx live
import { DataTable, DataTableColumn, DataTableColumnHeader } from '@/components/ui/data-table'

function EmptyState({ context }: { context: { query: string } }) {
  return (
    <div className="flex h-full items-center justify-center p-6 text-center text-sm text-muted-foreground">
      No results for <span className="mx-1 font-medium text-foreground">{context.query}</span>.
    </div>
  )
}

export default function App() {
  return (
    <DataTable
      EmptyPlaceholder={EmptyState}
      className="rounded-xl"
      context={{ query: 'USB dock' }}
      data={{ data: [], groups: [] }}
      style={{ height: 240 }}
    >
      <DataTableColumn field="name">
        <DataTableColumnHeader>Product</DataTableColumnHeader>
      </DataTableColumn>
    </DataTable>
  )
}
```

## Patterns

- Put shared UI state into `context`, not onto every row object.
- Use `EmptyPlaceholder` for search, filter, and first-run states.
- Prefer composition around the table over bolting unrelated state into cell renderers.
