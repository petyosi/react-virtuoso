`@virtuoso.dev/data-table` is not a drop-in replacement for `TableVirtuoso`. It keeps
virtualization built in, but moves table structure into explicit data binding and column
declarations.

## API mapping

| TableVirtuoso pattern  | Data-table equivalent                                          |
| ---------------------- | -------------------------------------------------------------- |
| `data` array           | `data={{ data: rows, groups: [] }}` or a `model`               |
| `itemContent`          | `DataTableColumn` plus `DataTableCell`                         |
| `fixedHeaderContent`   | `DataTableColumnHeader`                                        |
| grouped table rows     | `data.groups` plus `GroupHeaderCell`                           |
| fixed columns          | `sticky="left"` or `sticky="right"`                            |
| custom table pieces    | shadcn wrapper classes or `components` overrides               |
| programmatic scrolling | `engineRef` / `engineId` plus `scrollToRow$`                   |
| range callbacks        | `onRenderedDataChange`, `viewportRange$`, or `scrollLocation$` |

## Migration sequence

1. Move row loading into [Data Binding](/data-table/data-binding/).
2. Convert each visible table column into a [Column Definition](/data-table/column-definitions/).
3. Add grouped rows, sticky columns, resizing, reordering, or visibility as separate opt-in
   features.
4. Replace ref-based commands with [Table Instance Control](/data-table/features/table-instance-control/).
5. Move visual table styling into the shadcn wrapper or the customization slots.

## Example shape

```tsx
<DataTable data={{ data: rows, groups: [] }} style={{ height: 420 }}>
  <DataTableColumn field="name">
    <DataTableColumnHeader>Name</DataTableColumnHeader>
    <DataTableCell>{({ row }) => row.data.name}</DataTableCell>
  </DataTableColumn>
  <DataTableColumn field="status" sticky="right">
    <DataTableColumnHeader>Status</DataTableColumnHeader>
    <DataTableCell>{({ cellValue }) => String(cellValue)}</DataTableCell>
  </DataTableColumn>
</DataTable>
```

The important change is ownership: `TableVirtuoso` receives row renderers; data-table receives a
data source and column declarations, then feature modules add table behavior.
